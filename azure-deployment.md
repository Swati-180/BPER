# Azure Deployment Guide — ePER Platform

## Overview

The ePER platform deploys to two Azure services:
- **Backend**: Azure App Service (Node.js 20)
- **Frontend**: Azure Static Web Apps

---

## 1. Azure Resources

| Resource | Name | Notes |
|---|---|---|
| Resource Group | `rg-eper-prod` | All resources in same RG |
| App Service Plan | `asp-eper-backend` | B1 SKU minimum |
| App Service | `eper-backend` | Backend Node.js API |
| Static Web App | `eper-frontend` | React/Vite frontend |
| Key Vault (optional) | `kv-eper-secrets` | For secure env management |

---

## 2. Backend (App Service)

### Publish Code
```bash
# From backend/
zip -r release.zip . -x "node_modules/*"
az webapp deployment source config-zip \
  --resource-group rg-eper-prod \
  --name eper-backend \
  --src release.zip
```

### Application Settings (Environment Variables)
Set these in **Azure Portal → App Service → Configuration → Application Settings**:

| Key | Value |
|---|---|
| `MONGO_URI` | MongoDB Atlas connection string |
| `JWT_SECRET` | Strong random string (32+ chars) |
| `ANTHROPIC_API_KEY` | sk-ant-... |
| `PORT` | `8080` (Azure default) |
| `FRONTEND_URL` | `https://eper-frontend.azurestaticapps.net` |
| `WEBSITE_NODE_DEFAULT_VERSION` | `20.x` |

### Startup Command
```
node server.js
```

## 3. Frontend (Static Web Apps)

### staticwebapp.config.json (create in bper-frontend/ root)
```json
{
  "routes": [
    { "route": "/api/*", "rewrite": "/api/" },
    { "route": "/*", "rewrite": "/index.html" }
  ],
  "platform": {
    "apiRuntime": "node:20"
  }
}
```

### GitHub Actions CI/CD (auto-generated by Azure)
After creating Static Web App, Azure provides a `/.github/workflows/azure-static-web-apps-*.yml`.

Update the **build config**:
```yaml
- name: Build And Deploy
  uses: Azure/static-web-apps-deploy@v1
  with:
    azure_static_web_apps_api_token: ${{ secrets.AZURE_STATIC_WEB_APPS_API_TOKEN }}
    repo_token: ${{ secrets.GITHUB_TOKEN }}
    action: "upload"
    app_location: "bper-frontend"
    api_location: ""
    output_location: "dist"
    app_build_command: "npm run build"
```

### Build Steps
```bash
cd bper-frontend
npm install
VITE_API_URL=https://eper-backend.azurewebsites.net npm run build
```

## 4. MongoDB Atlas — Network Access

Add Azure outbound IPs to MongoDB Atlas IP Allowlist:
1. Go to Azure Portal → App Service → Properties → Outbound IP Addresses
2. Copy all IPs listed
3. Go to MongoDB Atlas → Network Access → Add IP Address → paste each IP

Or for simplicity in non-production: Allow all IPs (`0.0.0.0/0`)

---

## 5. CORS Configuration

Backend `server.js` reads `FRONTEND_URL` from env.
Set `FRONTEND_URL` to your Static Web App URL:
```
https://your-app-name.azurestaticapps.net
```

## 6. Seed the Database (one-time)

From your local machine with the MONGO_URI pointing to Atlas:
```bash
cd backend
node seed/seedEper.js
```
